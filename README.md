# NJ Sound Server

Use your Android phone as a wireless speaker and microphone for your PC.

## How it works

```
┌──────────────────────┐     PCM 48kHz (mono/stereo)     ┌──────────────────────┐
│  Linux (Python)      │ ◄──────── WebSocket ──────────► │  Phone Browser       │
│                      │                                  │                      │
│  PC audio output ────┼── audio stream (mono/stereo) ──►│  Phone speaker       │
│  PC audio input  ◄───┼── audio stream (mono) ──────────│  Phone microphone    │
└──────────────────────┘                                  └──────────────────────┘
```

- The server captures system audio and streams it to your phone in real-time
- Your phone's microphone captures your voice and streams it back to the PC
- Both directions happen simultaneously (full duplex)
- Optional stereo mode for PC→Phone audio

## Requirements

- **Server**: Linux with PulseAudio/PipeWire, Python 3.10+
- **Client**: Any smartphone with a modern browser (Chrome, Firefox, Safari)

## Quick start

```bash
git clone <repo-url> && cd nj-sound-server

# One-time setup (installs deps + generates SSL cert)
bash setup.sh

# Start the server
bash server.sh
```

The server will print a URL like `https://10.92.240.217:8765`. Open this URL in your phone's browser and accept the self-signed certificate warning.

## Usage

1. Open `https://SERVER_IP:8765` in your phone browser
2. Accept the self-signed certificate warning
3. Tap **Connect** — the browser will ask for microphone permission
4. Grant permission — PC audio plays through your phone speaker
5. Tap **Mic OFF** when you don't need to speak (prevents feedback)
6. Tap **Mic ON** to speak into the phone mic — your voice plays through your PC speakers
7. Tap **Disconnect** to stop

### Options

```bash
bash server.sh --port 8080              # Use a different port
bash server.sh --host 192.168.1.5       # Bind to specific IP
bash server.sh --device 2               # Use a specific audio input device
bash server.sh --stereo                 # Enable stereo PC→Phone audio
bash server.sh --list-devices           # Show available audio devices
bash server.sh --cert cert.pem --key key.pem  # Custom SSL certificate
bash server.sh --version                # Show version
```

| Option | Description |
|---|---|
| `--port PORT` | Server port (default: 8765) |
| `--host ADDR` | Bind address (default: 0.0.0.0) |
| `--device ID` | PulseAudio source device index |
| `--stereo` | Stream PC audio in stereo (default: mono) |
| `--list-devices` | List all available PulseAudio devices and exit |
| `--cert FILE` | SSL certificate path (auto-detects cert.pem) |
| `--key FILE` | SSL private key path (auto-detects key.pem) |
| `--version` | Show version number and exit |

## Audio quality

The server streams uncompressed 48kHz PCM audio:

- **Default**: Mono (1 channel), ~180 KB/s bandwidth
- **With `--stereo`**: Stereo (2 channels), ~360 KB/s bandwidth
- **Clip protection**: Samples are clipped to [-1.0, 1.0] before Int16 conversion — no distortion at high volumes
- **Jitter buffer**: The phone buffers 200ms of audio before starting playback, then uses a 4096-sample processing window (~85ms) for glitch-free playback
- **Smooth underrun handling**: When buffer runs empty (network jitter), audio fades out gracefully instead of cutting to silence

## Mic control

A **Mic ON/OFF** toggle appears when connected. When muted, the phone mic stops transmitting but the phone still plays PC audio — useful for preventing feedback when you're not speaking.

## HTTPS

The server serves over HTTPS with a self-signed certificate (auto-generated by `setup.sh`). This is required for microphone access on Android browsers. Accept the browser warning to proceed.

## Latency

- Typical round-trip: ~80-150ms (varies by WiFi and phone)
- 20ms frames at 48kHz for each direction
- 200ms jitter buffer on phone
- Phone's browser adds ~10-30ms buffer (unavoidable in Web Audio API)

## Limitations

- **Background**: The browser must stay in the foreground — Chrome suspends audio when the tab is backgrounded
- **WiFi only**: Comfortable on any WiFi network
- **Echo**: Keep the phone mic muted (Mic OFF) when you're not speaking to prevent feedback
- **Browser support**: Chrome/Firefox/Samsung Internet on Android; Safari on iOS (higher latency)

## Architecture

```bash
server/
├── audio.py           # Audio capture (PulseAudio monitor) + playback
├── handler.py         # WebSocket server — relays PCM frames
├── main.py            # HTTP/HTTPS server + entry point
├── static/index.html  # Web client (single HTML file)
├── cert.pem           # Self-signed SSL certificate (auto-generated)
├── key.pem            # SSL private key (auto-generated)
└── requirements.txt   # Python dependencies
```

The protocol is bidirectional PCM streaming over WebSocket:

| Mode | Frame size | Samples | Format |
|---|---|---|---|
| Mono (default) | 1920 bytes | 960 Int16 | 960 mono samples |
| Stereo (`--stereo`) | 3840 bytes | 1920 Int16 | 960 L/R pairs interleaved |

- Server sends system audio frames → phone plays them
- Phone sends mic audio frames (always mono) → server plays them
